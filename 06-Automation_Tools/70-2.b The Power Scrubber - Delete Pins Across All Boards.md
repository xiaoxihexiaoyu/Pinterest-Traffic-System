如果你有很多大的 Board 和大量的 Pin，这个 Automation 可以规模化地清理它们。

### 这个版本的功能

这个 Automation：

1. 遍历你账户中的每个 Board
2. 使用 Pinterest 内部书签系统每次获取 250 个 Pin
3. 筛选符合条件的 Pin：
   - 发布超过 90 天 并且
   - 终身展示数为 0
4. 删除每个匹配的 Pin
5. 重复处理直到所有 Pin 都被处理，每个 Board 最多处理 1,250 个 Pin（如果调整循环，可以更多）
6. 错误处理（如果 Pinterest API 触发太快，会暂停 100 秒）

它由分页驱动。它很强力。它在大型账户上运行效果极佳。

### 仅在需要时：更改默认参数

每个 Board 的最大 Pin 数

有两个参数定义了 Automation 可以处理的每个 Board 的最大 Pin 数

重复次数（默认值 = 5，范围 1...100） 决定每个 Board 将获取多少个 API 请求（页面）。

每次请求的 Pin 数（默认值 = 250，范围 1...250） 定义每次 API 调用检索多少个 Pin。

重要：值不得超过 250，这是 Pinterest API 的限制。

因此，默认情况下每个 Board 的最大 Pin 总数 = 5  250 = 1,250 个 Pin/Board。

如果你的 Board 有超过 1,250 个 Pin/Board，你必须增加 Repeats 到相应的值。

如果你的所有 Board 都明显少于这个数量，你可以将 Repeats 减少到较低的值。

例如，所有 Board 都少于 250 个 Pin，将 Repeats 设置为 1。场景将运行更快并使用更少的操作。

### Board 的最大数量

在"List Boards"模块中有一个参数可以调整 Board 的最大数量

默认值为 150，因此将处理已连接 Pinterest 账户的最多 150 个 Board。

如果存在更多 Board，你需要相应地提高此限制。

你可以将此值调整为 1 来测试场景。

### 这个版本的优点

- 深度清理你所有的 Board
- 可扩展适用于大型 Pinterest 账户
- 自动处理 Pinterest 的分页逻辑
- 包含错误处理（以防 Pinterest API 变慢）

### 限制

- 结构稍微复杂一些
- 只删除终身展示数为 0 的 Pin（默认）
- 在非常大的账户上运行可能需要更长时间
- 在非常大的账户上可能会超过操作数量（取决于你的 Make 计划）

你始终可以减少重复次数以使其运行更快并节省操作。

### 最佳使用场景

- 管理 100+ 个 Board
- 执行月度清理程序
- Pinterest VA、代理商或高级用户
- 任何想要完全掌控账户健康状况的人
