让 Automation 按照你的方式工作，而不意外删除你最好的 Pin！

警告：所有讨论的 Automation 都会永久删除你的 Pinterest 账户中的 Pin。一旦删除，Pin 无法恢复。更改过滤器会导致行为变化！更改和使用风险自负。我们强烈建议使用重复的场景和少量 Pin 或 Board 进行测试。

### 为什么过滤器很重要

过滤器是告诉 Automation 的方式：

"只删除满足这些确切条件的 Pin。"

默认情况下，Automation 很智能但很严格：

- Pin 必须超过 90 天 并且
- 必须有 0 终身展示数

这是一个可靠的基础。

但也许你想要：

- 更激进
- 更宽容
- 或基于其他指标（如点击数或保存数）进行过滤

### 在哪里找到过滤器

在你的场景中，查找名为"Run filter for each pin"和"Delete pin when passed the filter"的模块。在这两个模块之间的连接线上有一个小漏斗图标，那就是过滤器。

右键点击它 选择"Edit filter"/编辑过滤器

信息：在基于分页的版本中，你会看到有两个（相同的）过滤器模块：

- 一个用于第一批 Pin
- 一个用于后续的分页批次

重要：如果你更改了一个过滤器，你必须手动更新两个，否则 Automation 将会表现不可预测！

### 你可以更改什么

思维是你的极限 - 但保持简单愚蠢。

在你疯狂构建 12 层过滤器逻辑、嵌套条件和宇宙能量对齐之前：
- 停下来。

是的，Make.com 给了你疯狂的灵活性。

但你的逻辑越简洁，结果就越好（你的账户也越安全）。

### 了解两种类型的 Pin 指标

Pinterest 为每个 Pin 提供两个独立的指标块，"Lifetime"/终身 和 "90-Day"/90天。它们听起来相似，但在实践中非常不同：

1. lifetime_metrics / 终身指标

一个 Pin 存在的全程真相。

这个指标块包含自 Pin 发布以来的参与度总计，无论它有多老。

你会找到以下值：

- impression / 展示
- pin_click / Pin 点击
- outbound_click / 外部点击
- save / 保存
- reaction / 反应
- comment / 评论
- profile_visit / 资料访问

最适合：
- 检测从未表现的 Pin
- 保持常青内容活跃（如果它过去表现强劲）

限制：
- 不能告诉你参与度何时发生，可能是 2 年前

2. metrics_90d ( pin_metrics.90d ) / 90天指标

仅来自最近 90 天的参与度数据

这个指标块包含最近 90 天的参与度总计。它告诉你一个 Pin 是否仍然活跃，或者只是古老的死重。

你获得相同的核心指标：

- impression / 展示
- pin_click / Pin 点击
- outbound_click / 外部点击
- save / 保存
- reaction / 反应
- comment / 评论
- profile_visit / 资料访问

最适合：
- 检测曾经表现但现在已过时的 Pin
- 过滤失去动力的内容

限制：
- 如果你拥有季节性内容，使用可能很危险。

### 编辑前的安全提示

- 在开始实验之前导出当前 blueprint 的备份
- 在场景的副本上测试
- 用 Board 名称重命名每个克隆。这样保持有条理！
- 用少量 Pin 或单个 Board 测试
- 首先用 Log 模块替换"Delete Pin"模块
- 更新两个过滤器块（初始 + 分页路径）

### 回顾：智能过滤器策略

- 从默认逻辑开始
- 仅在你有信心时调整
- 在扩展之前在小账户上或使用版本 2.c 测试
- 记住：一旦 Pin 被删除 - 它就永远消失了
