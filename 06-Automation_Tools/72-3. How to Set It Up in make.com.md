从"我下载了文件"到"它正在删除 Pin"只需 10 分钟：

### 开始前你需要什么

- 一个 make.com 账户（免费或付费）
- 一个 Pinterest 商业账户
- 一个 Automation blueprint 文件（你将在下面获得）
- 几分钟的专注时间

### 分步设置：

步骤 1：登录 make.com

访问 make.com 并登录或注册。

如果你还没有账户，你可以在 60 秒内免费创建一个。

步骤 2：创建新场景

点击紫色的"+ "Create a new scenario"创建新场景"按钮：

你将看到一个空白画布。

步骤 3：导入 Automation blueprint

点击场景屏幕底部菜单栏中间的三个点（•••）。

选择： "Import Blueprint"/导入 Blueprint

然后选择你想使用的版本的 JSON 文件，例如 The Quick Fix – Simple Version (Limited to 250 pins - board).blueprint.json

导入后，模块将出现，布局精美。

步骤 4：连接你的 Pinterest 账户

点击任何 Pinterest 模块（例如"List Boards"或"Delete Pin"）并选择你想使用的 Pinterest 账户连接。

如果你是第一次在 Make 中使用 Pinterest，你必须点击"Add"创建新连接，然后你会被提示：

- 命名连接（提示：使用你的 Pinterest 用户名以便清晰）
- 通过 Pinterest 登录并授予权限

重要：连接后，你必须连接场景中的所有 Pinterest 模块。

步骤 5：测试你的 Pinterest 连接

右键点击 Pinterest 模块"List Boards"或"Get Board"

选择：
- "Run this module only"

如果你看到 绿色勾选且没有错误，你已连接。让我们开始吧。

步骤 6：保存并运行场景

点击 "Save"保存按钮（底部工具栏）

然后点击：
- "Run once"

现在观看它的运行。

你将看到 Pin 被处理、过滤 - 如果匹配，则被删除。

### 安全测试（可选但明智）

- 临时用 Log 模块或 Email 模块替换 Delete Pin 模块
- 运行测试并查看哪些 Pin 会被删除
- 一旦你有信心，重新启用删除

就是这样！

你正式运行了最强大的 Pinterest 维护工具之一，并且它 100% 在你的控制之下。
